Installation

First of all to run this application You will need Node.js and Gulp.js. You can find Nodejs installation for Linux/Windows/MacOs tutorial here.

I. Ubuntu installation:

1. Install Node.js ( if already installed, skip this step):

apt-get install nodejs

2. Install Gulp.js:

npm install -g gulp

3. Install Bower:

npm install -g bower

4. Inside the application folder run npm budle installation:

npm install

5. Run bower bundle installation:

bower install

 

II. MacOs installation:

1. First of all you need brew ( if already have it, skip this step ):

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

2. Install Nodejs using brew:

brew install nodejs

3. Install gulp.js:

npm install -g gulp

4. Install bower:

npm install -g bower

5. Inside the application folder run npm budle installation:

npm install

6. Run bower bundle installation:

bower install
 

 

 

 

 

 

Gulp.js

Why Gulp.js and not Grunt.js or npm ? Because it's faster!  and soooo much easier.

This application is desing with two most wanted and basic functions for building production or development application. Feel free to check the gruntfile.js for and improvement or personalization to You needs.

Development mode 

gulp serve

The most common CLI command, that prepare Your application for development with livereloading, image optimalization, javascript contacation or/and minification. This gulp task will run your application in the browser in few seconds time and will update it when you change something in you application. Really usefull, will prevent from raping that refresh button :) 

Production mode

gulp build

Second most usefull CLI command. This simply runs the same tasks that gulp serve will load with all the minification and concatanation and will copy the outputed application in build folder where ready-for-production app will be.

Go public!

gulp deploy

This command allows You to deploy your application on remote server via FTP. It's super easy, just create a file named

.ftppass

in root folder (next to bower.json and gulpfile.js ). Pass these script inside ftppass,

{
"devAuth": {
   "user": "YOUR_FTP_USER",
   "pass": "YOUR_FTP_PASS"
   }
}

Replace YOUR_FTP_USER and YOUR_FTP_PASS with yours FTP server credentials, and you are done. Run it! It will build Your application in production enviroment and it will push your outputed app to remote server.

 

 

 

Structure

I. Application structure

The application is stored in /app directory. As You can see there are 3 items:

1. Assets - this is where we store all the static files like stylesheets, javascripts, images or fonts.

/app/assets

2. Partials - this is where we keep all the partials - they are loaded by Angular.js (if needed)

/app/partials

3. index.html - this is the root .html file.

 

1. Assets

This folder contains all the assets used in this application like css, js, fonts, images. 

/app/assets/css
/app/assets/js
/app/assets/fonts
/app/assets/img
 

2. Partials

/app/partials/admin
/app/partials/bootstrap
/app/partials/charts
... etc

Partials folder, as the name states, contains all the partial html files like modals, sidebars, mail etc etc, There are categorized so it should be easy to find the file that you need. There are pulled from server if needed for example if you are on the mail page the application will pull the index.html, sidebar.html and subside.html folder: 

/app/partials/mail/
It because of the lazyLoad module (called ocLazyLoad for for Angular.js) we can get only what we currently need. That makes the aplication very lightwaight and super fast.

3.1. Loading dynamic content

<div ui-view></div>

This is an root view element from ui-routers directive where will all the top-level content renders. Like sidebar partial or top menu partial and even main page partial it depends on the route that You the user will change browsing the application. This is very importat, without this container the html will not load and you will see a blank page.

3.2. Enviroment variables

<!-- @if NODE_ENV='DEVELOPMENT' -->

and 

<!-- @if NODE_ENV='PRODUCTION' -->

When You run gulp serve command the application is in Development mode - that means that it's not production ready, or in other words all the CSS and JS files are not concataneted and minified. Also you application that is pre-build in development mode is in .tmp directory.

These if statments simply removed or show inner html based on current enviroment ex. if Production mode is enabled the all the scripts and css files will be present in outputed index.html file. 

3.3. Minification and contacation

 <!-- build:js path/and/file -->

These comnent is for gulp.js application only. It will not be showed in outputed index.html in any case. This simply tells gulp.js that all this css/js files in this wrapped comment will be preproccesed and outputed in a sigle file ex. /path/to/file
